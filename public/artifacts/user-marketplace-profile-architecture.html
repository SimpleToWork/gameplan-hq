How a user works across marketplaces and profiles

Three entities in a tiered many-to-many. A user reaches many profile-marketplaces; a profile-marketplace holds many users. Everything downstream — billing, routing, permissions — hangs off that join.

05

Who can do what

Role lives on the user_profile row (user_type). It's per profile-marketplace, not global.

superuser
  • The profile creator defaults here
  • Can invite anyone
  • Can demote another superuser to admin / user
admin
  • Can invite users
  • Can promote only up to admin — never superuser
user
  • The default for an invited user
  • Cannot invite or promote anyone
INVARIANT  Every profile keeps at least one superuser at all times.
06

Sessions & onboarding state

session_details

One row per browser tab / connection. It records which profile_id + marketplace_id that tab is currently logged into — so two tabs can sit in two different pairings at once. This is what backs the toggle above.
