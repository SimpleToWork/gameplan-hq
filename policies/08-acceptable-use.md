# Acceptable Use Policy

**Policy Number:** SEC-008  
**Version:** 1.0  
**Effective Date:** 2026-06-25  
**Owner:** CEO / Security Officer  
**SOC 2 Criteria:** CC6.1, CC1.1, CC1.2

---

## 1. Purpose

Define the acceptable use of MerchantsBI's information systems, devices, and data to protect
company assets and maintain a secure working environment.

## 2. Scope

All employees, contractors, and third parties using MerchantsBI systems, devices, or data —
including company-provided and personally-owned devices used for work (BYOD).

## 3. General Principles

- Systems and data are provided for business purposes. Incidental personal use is permitted if
  it does not interfere with work, violate policy, or create security risk.
- Users are responsible for activities conducted under their credentials.
- Privacy on company systems is limited — MerchantsBI may monitor usage for security purposes.

## 4. Permitted Use

- Accessing Gameplan HQ, Firebase, Vercel, GitHub, and other work tools for authorized job duties.
- Using company email and communication tools for business communication.
- Storing work files in approved company systems (Google Drive, Firestore).
- Using AI tools (Claude, ChatGPT, etc.) for productivity, subject to the AI Use guidelines below.

## 5. Prohibited Use

The following are **prohibited** on MerchantsBI systems and accounts:

- Accessing, storing, or transmitting illegal content.
- Using MerchantsBI systems to attack, probe, or disrupt third-party systems without explicit
  written authorization.
- Sharing MerchantsBI credentials, API keys, or access tokens with unauthorized individuals.
- Committing secrets, credentials, or API keys to source code repositories.
- Installing unauthorized software on company devices that could introduce malware.
- Using work accounts for personal side projects or businesses that compete with MerchantsBI.
- Circumventing security controls (disabling MFA, using VPN to bypass access restrictions, etc.).
- Sharing or publishing Restricted or Confidential data publicly without authorization.

## 6. AI Tool Use

- AI tools may be used to assist with work tasks; treat them as you would a third-party vendor.
- **Do not paste Restricted data** (API keys, credentials, customer PII, confidential financials)
  into external AI tools unless the tool has a signed DPA and is on the approved vendor list.
- AI-generated code must be reviewed for security before merging to production.
- AI outputs are not a substitute for human judgment; the employee remains responsible for
  accuracy and quality.

## 7. Device Security

- Devices used for work must:
  - Have full-disk encryption enabled (FileVault on Mac, BitLocker on Windows).
  - Run current OS and software updates (within 30 days of critical patch release).
  - Have screen lock set to ≤ 5 minutes idle.
  - Not be left unlocked and unattended in public places.
- Lost or stolen devices must be reported to the Security Officer within **2 hours**.

## 8. Remote Work

- Use only trusted networks (home or cellular) for work involving Restricted data.
- Do not access production systems on open public Wi-Fi unless connected through a VPN.
- Screens displaying Confidential data should not be visible to unauthorized individuals.

## 9. Violations

Violations may result in:
- Revocation of access.
- Disciplinary action up to and including termination.
- Legal action where applicable.

Employees who suspect a policy violation should report it to the Security Officer.
